# Картотека — HTTP клиент и сервер для сбора информации из интернета

Курсовая работа: клиент-серверное веб-приложение, которое по ключевому слову находит
список источников, скачивает выбранную страницу через сервер с отображением
прогресса в реальном времени (WebSocket) и сохраняет результат в `LocalStorage`
браузера для последующего чтения офлайн.

## Стек

- **Сервер:** Node.js, Express (REST API), `ws` (WebSocket для прогресса загрузки)
- **Клиент:** чистый JavaScript (без фреймворков), `fetch` + `WebSocket`, `LocalStorage`
- Зависимостей кроме `express` и `ws` нет — это сделано намеренно, чтобы код было легко читать и проверять

## Как это работает (алгоритм)

1. Клиент вводит ключевое слово → `GET /api/keywords/:word` → сервер возвращает список URL.
2. Клиент выбирает один URL из списка → `POST /api/download` (тело: `{ url, clientId }`).
3. Сервер сразу отвечает `{ jobId }` и в фоне начинает скачивать страницу по этому URL.
4. По мере скачивания сервер шлёт клиенту через WebSocket сообщения о прогрессе:
   `{ type: 'progress', jobId, loaded, total, progress }`.
5. По завершении сервер шлёт `{ type: 'done', content, size, contentType, ... }`,
   клиент сохраняет результат в `LocalStorage`.
6. Список сохранённого контента доступен в колонке «Архив» — можно открыть и читать
   офлайн (контент уже лежит в браузере, повторных запросов к серверу не требуется).

## Структура проекта

```
web-collector/
├── server/
│   ├── index.js          # точка входа: Express-роуты + инициализация WebSocket
│   ├── keywordsStore.js  # чтение словаря "ключевое слово -> [URL]" из data/keywords.json
│   ├── downloader.js     # скачивание контента по URL с колбэком прогресса (http/https, редиректы)
│   └── wsManager.js      # реестр WebSocket-подключений по clientId, рассылка сообщений
├── public/
│   ├── index.html        # разметка интерфейса (поиск / загрузка / архив)
│   ├── style.css          # стили
│   └── app.js             # вся клиентская логика (fetch, WebSocket, LocalStorage)
├── data/
│   └── keywords.json     # словарь ключевых слов и URL (можно редактировать без перезапуска кода — только перезапустить сервер)
├── package.json
└── README.md
```

## Установка и запуск (локально)

Требуется **Node.js версии 18 или новее** (использует `crypto.randomUUID`, встроенный `fetch`).

```bash
# 1. Клонировать репозиторий
git clone <ссылка-на-репозиторий>
cd web-collector

# 2. Установить зависимости
npm install

# 3. Запустить сервер
npm start
```

После запуска сервер поднимется на `http://localhost:3000`.
Откройте этот адрес в браузере — это и есть интерфейс клиента (сервер раздаёт статику из `public/`).

Порт можно изменить переменной окружения `PORT`:

```bash
PORT=8080 npm start
```

## Как пользоваться

1. Введите ключевое слово (или нажмите на одну из подсказок-чипов под полем ввода —
   доступны: `новости`, `наука`, `программирование`, `погода`, `спорт`, `космос`)
   и нажмите «Найти».
2. В списке найденных источников кликните на нужный URL — начнётся загрузка,
   справа появится прогресс-бар с размером и процентом.
3. После завершения контент автоматически сохранится в колонку «Архив» (хранится
   в `LocalStorage` браузера — переживёт перезагрузку страницы).
4. Кликните на любую запись в архиве, чтобы открыть просмотр содержимого
   (доступно полностью офлайн — повторный запрос к серверу не делается).
   Кнопка «Исходный код» откроет сырой HTML в отдельной вкладке, «Удалить» —
   уберёт запись из архива.

## Обработка ошибок

Приложение обрабатывает и показывает пользователю в понятном виде:

- пустое / не найденное ключевое слово (`404` с пояснением);
- отсутствие соединения с сервером (REST или WebSocket недоступны);
- недоступный хост, таймаут запроса, некорректный URL;
- HTTP-ошибки целевого сайта (4xx/5xx), слишком большой размер контента (защита от перегрузки, лимит 15 МБ);
- разрыв WebSocket-соединения — клиент автоматически переподключается каждые 3 секунды,
  индикатор в шапке страницы показывает текущий статус связи;
- переполнение `LocalStorage` браузера.

## Добавление новых ключевых слов

Отредактируйте `data/keywords.json` — формат `{ "слово": ["url1", "url2", ...] }`.
После изменения файла сервер нужно перезапустить (`npm start`).

## Публичный доступ (по желанию)

Само задание просит, чтобы рабочий результат был доступен публично без обязательного
локального развёртывания. Проще всего это сделать бесплатно через **Render**:

1. Зарегистрируйтесь на [render.com](https://render.com) и подключите GitHub-репозиторий с этим проектом.
2. Создайте **Web Service**, окружение — **Node**.
3. Build Command: `npm install`
4. Start Command: `npm start`
5. Render сам пробросит переменную `PORT` — сервер уже её поддерживает (см. `server/index.js`).

После деплоя Render выдаст публичный URL вида `https://<имя-проекта>.onrender.com` —
именно его можно указать в качестве «работающего результата» при сдаче задания.
